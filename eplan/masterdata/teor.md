# Керування основними даними в Eplan: теоретична частина 

### 6.1. Основні дані проєкту і системи

#### 6.1.1. Поняття «основні дані»

Під поняттям «основні дані» в EPLAN розуміють усі дані, за допомогою яких створюється проєкт. Іншими словами, це бібліотечні елементи, що використовуються під час розроблення проєкту.

В EPLAN до основних даних належать:

- символи;
- рамки;
- форми для звітів;
- макроси;
- шаблони проєктів і базові проєкти;
- база даних іноземних мов;
- дані виробів, включаючи означення пристроїв тощо;
- означення функцій;
- графічні файли;
- файли DXF / DWG;
- специфічна для EPLAN база даних прав.

В EPLAN розрізняють основні дані системи та основні дані проєкту.

#### 6.1.2. Основні дані системи

Основні дані системи – це основні дані для всіх проєктів, які зберігаються в централізованому пулі основних даних. Їх можна змінювати за допомогою функцій обробки основних даних.

Основні дані системи розміщуються у вигляді файлів у відповідних каталогах компанії. Розташування задається в налаштуваннях:

Параметри → Налаштування → Користувачі → Керування → Каталоги

Крім того, під час інсталяції оригінальні файли створюються також у папці Eplan.

#### 6.1.3. Основні дані проєкту

Основні дані проєкту – це використані та збережені в проєкті основні дані. У проєкті немає прямого посилання на основні дані системи. Під час проєктування завжди обробляється копія, специфічна для цього проєкту.

Тобто при використанні в проєкті основних даних системи вони копіюються до проєкту і стають незалежними від системних даних. Наприклад, якщо у проєкті потрібно використати символи з певної бібліотеки, ця бібліотека копіюється до проєкту і стає частиною його основних даних.

Основні дані проєкту зберігаються у папці проєкту у вигляді файлів. Для очищення від невикористаних даних застосовується стиснення проєкту:

Проєкт → Організувати → Стиснути

#### 6.1.4. Поняття синхронізації основних даних

Хоча після копіювання основні дані проєкту стають незалежними від системних, їх можна синхронізувати.

Синхронізація проєкту – це процес зміни основних даних проєкту відповідно до основних даних системи або навпаки.

Наприклад, якщо в базі даних виробів змінено властивості певного виробу, це не впливає на його копію в проєкті. Проте під час синхронізації властивості виробу в проєкті переписуються даними із системної бази даних виробів.

У деяких випадках, наприклад після оновлення EPLAN, необхідно оновити основні дані системи з оригінальних файлів у папці Eplan, тобто виконати синхронізацію основних даних системи.

#### 6.1.5. Способи синхронізації

Основні дані проєкту можна синхронізувати:

- під час відкриття проєкту;
- під час резервування або відновлення;
- за запитом.

Для автоматичної синхронізації під час відкриття потрібно змінити налаштування:

Параметри → Налаштування → Проєкти → «Ім’я проєкту» → Керування → Загальне

| Опція                                             | Якщо активована                                              |
| ------------------------------------------------- | ------------------------------------------------------------ |
| Синхронізувати основні дані проєкту при відкритті | Під час відкриття синхронізуються лише ті дані, для яких встановлено опцію «Синхронізувати». |
| Синхронізувати рамки                              | Рамки синхронізуються під час відкриття за умови активної загальної синхронізації. |
| Стандартна рамка                                  | Під час відкриття вироби проєкту синхронізуються з базою даних виробів. Якщо дані застаріли, з’являється запит. |

Синхронізація за запитом:

Сервісні програми → Основні дані → Синхронізувати поточний проєкт

Синхронізація основних даних системи:

Сервісні програми → Основні дані → Синхронізувати основні дані системи

Увага. Синхронізація замінює дані проєкту однойменними системними даними. Виконуйте її лише за потреби.

### 6.2. Керування різними типами основних даних проєкту

#### 6.2.1. Керування бібліотеками символів

Символи об’єднуються в бібліотеки символів, де вони обробляються і керуються. Бібліотеки додаються до проєкту через меню:

Параметри → Налаштування → Проєкти → «Ім’я проєкту» → Керування → Бібліотеки символів

Опція «Синхронізувати» визначає, чи буде бібліотека синхронізуватися під час відкриття проєкту.

«Префікс» використовується для ідентифікації бібліотеки у списках вибору символів. За замовчуванням використовується перша літера назви бібліотеки.

«Стандартний варіант» визначає варіант символу (кут повороту, A–G) під час вставлення.

Невикористані бібліотеки можна видалити через механізм стиснення проєкту. Бібліотеки SPECIAL і GRAPHICS не видаляються навіть якщо не використовуються.

#### 6.2.2. Керування рамками

Рамки обробляються як окремі файли і додаються до проєкту при першому використанні. Якщо створюється сторінка з рамкою, якої ще немає в проєкті, вона зберігається в основних даних проєкту.

Навіть якщо рамку змінити в налаштуваннях сторінки, вона залишається в основних даних проєкту. Перелік рамок можна переглянути у вікні синхронізації проєкту.

Невикористані рамки видаляються через стиснення проєкту.

Рамка за замовчуванням задається в меню:

Параметри → Налаштування → Проєкти → «Ім’я проєкту» → Керування → Сторінки → Стандартна рамка

#### 6.2.3. Керування формами

Форми обробляються як окремі файли і додаються до проєкту при першому використанні.

Форми для кожного типу звіту задаються в меню:

Параметри → Налаштування → Проєкти → «Ім’я проєкту» → Звіти → Вивід на сторінках

Після першого формування звіту відповідна форма копіюється в основні дані проєкту.

Перелік форм можна переглянути у вікні синхронізації проєкту. Невикористані форми видаляються через стиснення проєкту.

#### 6.2.4. Керування даними виробів

Окрім загальної бази даних виробів, існує база даних виробів проєкту, де зберігаються використані в проєкті вироби.

Дані виробу копіюються до проєкту під час вибору виробу з бази даних або якщо він міститься у макросі.

Перелік виробів у проєкті та їх синхронізацію можна переглянути через меню:

Сервісні програми → Виріб → Синхронізувати поточний проєкт

У цьому вікні можна також оновити або додати вироби до бази даних із даних проєкту.

Оновлення застарілих даних виробів у проєкті:

Сервісні програми → Виріб → Оновити поточний проєкт

### 6.3. Редактори основних даних системи

Для зміни основних даних системи використовуються спеціальні редактори. Вони змінюють саме системні дані, а не дані проєкту.

Тому якщо потрібно змінити основні дані у проєкті, спочатку вносяться зміни в основні дані системи, а потім виконується синхронізація проєкту.
